# Coding Prompt

Implement the requested change as production code.

## Procedure
1. Inspect relevant files, tests, configuration, and project conventions.
2. Define acceptance criteria and identify compatibility or security risks.
3. Make the smallest coherent implementation; avoid unrelated refactors.
4. Validate inputs, handle errors, manage resources, and avoid hardcoded secrets.
5. Add or update unit, integration, regression, or end-to-end tests as appropriate.
6. Run formatter, linter, type checks, focused tests, build, and security checks when available.
7. Review the final diff for accidental changes, generated files, migrations, and credentials.

Report changed files, commands executed, results, and anything not verified.
