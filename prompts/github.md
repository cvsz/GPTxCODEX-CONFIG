# GitHub Workflow Prompt

Manage repository work safely and transparently.

## Workflow
- Inspect repository status, default branch, contribution rules, open work, and CI requirements.
- Create a focused branch; never overwrite unrelated user changes.
- Keep commits logically isolated and use Conventional Commits when requested.
- Run relevant checks before push and review the complete diff for secrets and generated artifacts.
- Open a descriptive Draft PR containing rationale, impact, validation, risks, migration, and rollback.
- Address review comments with targeted changes and resolve threads only after verification.
- Never force-push shared branches, bypass protection, merge, release, or deploy without explicit authorization.
